# Portfolio Project Guidelines

This document serves as the foundational mandate for this portfolio project. All design and development decisions must align with these standards.

## 1. Content Strategy: Niche vs. Breadth

**Rule:** Lead with your niche.
- Identify as a specialist (e.g., "React Developer focused on performance") rather than a generalist.
- Avoid listing every possible skill in the hero section.
- **Project Selection:** Showcase 3–5 high-impact projects that reinforce your niche.

## 2. Page Structure & Ordering

The portfolio must follow this specific order:
1.  **Hero:** Name + Role + One-liner pitch + CTA + Availability Badge.
2.  **About:** Short personal story + Motivation + Optional photo.
3.  **Projects:** 3–5 Case-study cards (Problem, Tech, Outcome, Links).
4.  **Skills:** Grouped logically (Languages, Frameworks, Tools, Cloud). Max 10–15 strong skills.
5.  **Blog (Optional):** Even a single post on technical learning.
6.  **Contact:** Email + LinkedIn + GitHub + Downloadable PDF Resume.

## 3. Design System & Aesthetics

### Typography
Use one of the following approved font pairings. **Avoid default fonts (Inter, Roboto, Arial).**
- **Editorial:** Fraunces + DM Sans (Distinctive, warm).
- **Tech-forward:** Syne + Manrope (Bold geometric).
- **Professional:** Playfair Display + Lato (Classic contrast).
- **Coder Aesthetic:** DM Mono + DM Sans (Unique dev vibe).

### Frontend Aesthetics
- **Colors:** Use ONE strong accent color on a neutral base (near-white or deep dark).
- **Layout:** Single-page scrollable, generous whitespace, mobile-first responsiveness.
- **Motion:** Subtle scroll-triggered fade-ins and hover states only. **NO parallax overload.**

## 4. Mandatory "MVP" Features

These features are non-negotiable for project completion:
- **Availability Signal:** A visible "Open to work" badge or status (e.g., "Available from July 2025").
- **Impact Numbers:** Quantify project outcomes (e.g., "Reduced load time by 40%").
- **Growth Mindset:** Each project description must include a "What I learned" section.
- **Performance:** Load speed must be under 2 seconds (Score 90+ on Lighthouse).
- **CTA Above the Fold:** A "Hire Me" or "Let's Talk" button must be visible in the Hero section.
- **AI Integration:** Inclusion of an AI-assisted interactive element/chatbot to guide recruiters.

## 5. Traps to Avoid
- ✗ Purple gradients on white.
- ✗ Dark mode only (ensure accessibility).
- ✗ Skill bars/percentages (they are arbitrary).
- ✗ Cluttered layouts without whitespace.
